# Early Access Onboarding Guide

## Audience

New contributors and operators supporting Foundation through Early Access Live.

## Onboarding Steps

1. Read [spec index](../specs/INDEX.md)
2. Review PRD-01 through PRD-04
3. Review platform technical design and API contracts
4. Learn role model responsibilities:
   - Planner: milestone/task planning
   - Builder: scoped implementation
   - Reviewer: quality and risk verification
   - Operator: deploy/monitor/incident handling
5. Execute a dry-run workflow in command hub
6. Review runbook and release process docs

## Contributor Expectations

- Keep changes incremental and reversible
- Maintain API/event contract compatibility
- Update documentation in same cycle as behavior changes
